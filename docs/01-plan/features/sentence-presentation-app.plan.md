# sentence-presentation-app - 계획 문서

> 버전: 1.0.0 | 작성일: 2026-05-30 | 상태: 완료
> 프로젝트 레벨: Starter

---

## 1. 개요

### 1.1 목적

영어 교사가 수업 중 영어 지문을 붙여넣으면, 앱이 지문을 문장 단위로 나누고 학생들이 보기 좋은 프리젠테이션 화면으로 크게 보여주는 교실용 웹 앱을 만든다.

이 앱은 읽기, 듣기, 따라 읽기, 해석, 문법 설명, 내용 이해 활동에서 학생들이 한 번에 한 문장에 집중할 수 있도록 돕는다.

### 1.2 배경

영어 수업에서는 긴 지문을 교실 화면이나 전자칠판에 띄워야 하는 경우가 많다. 하지만 문단 전체를 한 번에 보여주면 교실 뒤쪽에 앉은 학생은 글자를 읽기 어렵고, 교사가 특정 문장만 설명하기도 불편하다.

이 기능은 교사가 PowerPoint를 따로 만들지 않아도 붙여넣은 지문을 바로 문장별 발표 화면으로 바꿔준다.

## 2. 목표

### 2.1 주요 목표

- [ ] 교사가 영어 지문을 앱에 붙여넣거나 직접 입력할 수 있다.
- [ ] 앱이 지문을 자동으로 문장 단위 발표 슬라이드로 나눈다.
- [ ] 교사가 수업 중 이전 문장과 다음 문장으로 쉽게 이동할 수 있다.
- [ ] 문장 텍스트는 교실 맨 뒤 학생도 읽을 수 있도록 매우 크고 선명하게 표시된다.
- [ ] 앱은 각 문장을 주요소와 종요소로 나누어 계층적으로 보여준다.
- [ ] 주요소는 맨 윗줄에 진하게 표시하고, 종요소는 그 아래줄에 약간 작은 글씨로 표시한다.
- [ ] 발표 화면에서 주요소는 한 줄에 모두 표시한다.
- [ ] 발표 화면에서 주요소는 라벨 없이 색으로 구분하고, 색상 범례를 화면 위쪽에 표시한다.
- [ ] 발표 화면에서 종요소는 성분 하나당 한 줄로 표시한다.
- [ ] 종요소는 문장 안 원래 위치를 기준으로 해당 줄에서 가운데 정렬한다.
- [ ] 발표 화면 글씨체는 `Pretendard`를 사용한다.
- [ ] 교사가 글자 크기, 화면 테마, 정렬 같은 기본 표시 설정을 조절할 수 있다.
- [ ] 앱은 로그인이나 서버 설정 없이 브라우저에서 바로 실행된다.

### 2.2 이번 버전에서 하지 않는 것

- 사용자 계정, 로그인, 클라우드 동기화는 만들지 않는다.
- 데이터베이스나 백엔드 API는 만들지 않는다.
- 첫 버전에서는 자동 번역이나 AI 설명 기능을 넣지 않는다.
- 첫 버전에서는 PowerPoint 파일 내보내기를 넣지 않는다.
- 학생 답변 수집이나 퀴즈 기록 기능은 만들지 않는다.

## 3. 범위

### 3.1 포함 범위

- 교사용 지문 입력 영역
- 마침표, 물음표, 느낌표, 닫는 따옴표를 기준으로 한 기본 영어 문장 분리
- 한 화면에 한 문장을 크게 보여주는 발표 모드
- 문장 성분 분석 표시:
  - 주요소: 주어, 동사, 보어, 목적어
  - 종요소: 형용사어, 부사어
  - 주요소는 상단 줄에 진하게 표시
  - 종요소는 하단 줄에 약간 작은 글씨로 표시
  - 동일한 문장 성분은 중간에서 줄바꿈되지 않도록 표시
- 교사가 자동 분석 결과를 드래그앤드롭으로 수정하는 기능
  - 같은 줄 안에서 성분 순서 변경
  - 주요소 줄과 종요소 줄 사이의 성분 이동
  - 잘못 분류된 성분의 역할 변경
- 이전, 다음, 처음, 마지막 문장 이동 버튼
- 수업 중 빠르게 조작할 수 있는 키보드 조작
- 기본 표시 설정:
  - 글자 크기
  - 밝은 테마 / 어두운 테마
  - 텍스트 정렬
- 현재 위치 표시 예: `3 / 12`
- 입력값이 없을 때 안내 메시지와 검증 처리
- HTML, CSS, JavaScript 기반의 Starter 레벨 정적 웹 앱 구현

### 3.2 제외 범위

- 백엔드 저장소
- 여러 기기 사이의 실시간 교실 동기화
- PDF, Word, PowerPoint 파일 가져오기
- PPTX 파일 내보내기
- AI 기반 문장 단순화, 번역, 어휘 추출
- 학생 계정이나 교사용 대시보드

## 6. 발표 화면 가시성 기준

- 발표 문장은 기본값부터 매우 크게 보여야 한다.
- 글자 크기는 수업 환경에 맞게 더 크게 조절할 수 있어야 한다.
- 문장 영역은 화면 중앙에 배치하되, 너무 좁은 카드 안에 가두지 않는다.
- 교실 뒤쪽에서도 읽기 쉽도록 장식보다 선명한 글자, 넓은 여백, 높은 대비를 우선한다.
- 긴 문장은 줄바꿈되어도 자연스럽게 읽히도록 최대 너비와 줄 간격을 조정한다.
- 발표 모드에서는 불필요한 설명 문구를 줄이고, 문장 자체가 화면의 중심이 되게 한다.
- 주요소 줄은 발표 화면에서 가장 먼저 눈에 들어오도록 굵고 크게 표시한다.
- 종요소 줄은 주요소보다 작게 표시하되, 교실 뒤쪽에서도 읽을 수 있는 크기를 유지한다.
- 문장 성분 하나가 여러 단어로 이루어진 경우에도 해당 묶음 내부에서는 줄바꿈이 일어나지 않게 한다.

## 7. 문장 성분 표시 기준

### 7.1 성분 구분

- 주요소는 문장의 핵심 구조를 이루는 성분이다:
  - 주어
  - 동사
  - 보어
  - 목적어
- 종요소는 주요소를 꾸미거나 문장 의미를 보충하는 성분이다:
  - 형용사어
  - 부사어

### 7.2 화면 배치

- 첫 번째 줄에는 주요소를 배치한다.
- 첫 번째 줄에는 주요소를 모두 한 줄로 배치한다.
- 주요소에는 라벨을 붙이지 않고 색상으로 구분한다.
- 주요소 색상 범례는 발표 화면 위쪽에 배치한다.
- 종요소는 주요소 아래에 성분 하나당 한 줄로 배치한다.
- 종요소가 1개면 총 2줄, 종요소가 2개면 총 3줄이 된다.
- 종요소는 문장 안 위치를 기준으로 해당 줄에서 가운데 정렬한다.
- 주요소는 종요소보다 더 크고 굵게 표시한다.
- 종요소는 주요소보다 약간 작은 글씨로 표시하되, 가독성을 해치지 않는다.
- 성분 이름은 교사가 문법 설명을 할 수 있도록 함께 표시할 수 있다.

### 7.3 줄바꿈 규칙

- 하나의 문장 성분은 하나의 묶음으로 표시한다.
- 같은 문장 성분 안에 있는 단어들은 중간에서 줄바꿈되지 않아야 한다.
- 화면이 좁아 줄바꿈이 필요하면 성분 묶음 사이에서만 줄바꿈한다.
- 성분 묶음이 너무 길 경우에는 글자 크기를 줄이기보다 교사가 볼 수 있는 경고나 보조 표시 방식을 우선 검토한다.

### 7.4 분석 정확도 기준

- 첫 버전에서는 완벽한 영어 문법 분석보다 수업에 바로 쓸 수 있는 명확한 표시를 우선한다.
- 자동 분석이 애매하거나 틀린 문장은 교사가 수업 전 직접 수정할 수 있어야 한다.
- 교사는 드래그앤드롭으로 성분 묶음의 위치를 바꿀 수 있어야 한다.
- 교사는 주요소 줄과 종요소 줄 사이로 성분 묶음을 옮길 수 있어야 한다.
- 교사는 필요하면 성분 묶음의 역할 이름을 바꿀 수 있어야 한다.
- 분석 결과는 학생에게 혼동을 주지 않도록 성분 구분과 표시 규칙을 일관되게 유지한다.

### 7.5 교사 수정 규칙

- 자동 분석 결과는 최종 답이 아니라 교사가 수정할 수 있는 초안으로 다룬다.
- 드래그앤드롭 중에도 성분 묶음 내부 단어는 분리되지 않아야 한다.
- 성분 묶음을 이동해도 줄바꿈 금지 규칙은 유지되어야 한다.
- 수정한 결과는 현재 발표 세션 안에서 문장을 넘겨도 유지되어야 한다.
- 교사가 실수로 이동했을 때 되돌릴 수 있는 방법을 설계 단계에서 검토한다.

## 8. 성공 기준

- [ ] 교사가 문단을 붙여넣고 발표 모드를 시작할 수 있다.
- [ ] 앱이 일반적인 영어 지문을 문장별 슬라이드로 나눈다.
- [ ] 각 문장은 주요소와 종요소로 구분되어 표시된다.
- [ ] 주요소는 맨 윗줄에 굵게 표시되고, 종요소는 아래줄에 약간 작은 글씨로 표시된다.
- [ ] 발표 화면의 주요소는 한 줄에 모두 표시된다.
- [ ] 발표 화면의 주요소에는 라벨이 없고 색상 범례가 화면 위쪽에 표시된다.
- [ ] 종요소는 성분 하나당 한 줄로 표시된다.
- [ ] 동사와 목적어 사이의 부사어는 종요소 줄에서 동사와 목적어 사이를 기준으로 정렬된다.
- [ ] 같은 문장 성분에 속한 단어 묶음은 중간에서 줄바꿈되지 않는다.
- [ ] 교사는 잘못 분석된 성분 묶음을 드래그앤드롭으로 재배치할 수 있다.
- [ ] 교사는 주요소와 종요소 줄 사이로 성분 묶음을 이동할 수 있다.
- [ ] 이전/다음 버튼과 키보드 조작이 정상 작동한다.
- [ ] 발표 텍스트가 교실 대형 화면에 적합하게 매우 크게 보인다.
- [ ] 발표 화면에 `Pretendard` 글씨체가 적용된다.
- [ ] 표시 설정 변경이 발표 화면에 즉시 반영된다.
- [ ] 앱을 브라우저에서 로컬로 열 수 있다.
- [ ] 빈 입력, 짧은 지문, 긴 지문, 따옴표가 포함된 문장에 대한 기본 수동 QA를 통과한다.

## 9. 사용자 흐름

1. 교사가 앱을 연다.
2. 교사가 영어 지문을 입력 영역에 붙여넣는다.
3. 교사가 발표 시작 버튼을 누른다.
4. 앱이 첫 번째 문장을 문장 성분별로 분석해 발표 화면으로 보여준다.
5. 분석이 잘못된 경우 교사가 성분 묶음을 드래그앤드롭으로 옮긴다.
6. 교사가 버튼이나 키보드로 문장을 넘긴다.
7. 필요하면 글자 크기나 테마를 조절한다.
8. 교사가 편집 화면으로 돌아가 지문을 수정하거나 교체한다.

| 위험 요소 | 영향 | 가능성 | 대응 |
|-----------|------|--------|------|

| 매우 긴 문장이 발표 화면을 넘칠 수 있다. | 높음 | 중간 | 반응형 글자 크기, 최대 너비, 줄바꿈, 넓은 줄 간격을 적용한다. |

| 터치 화면과 마우스 환경에서 드래그 조작감이 다를 수 있다. | 중간 | 중간 | Pointer Events 기반 조작과 큰 드래그 핸들을 설계 단계에서 검토한다. |

## 12. 구조 고려사항

- `index.html`, `css/style.css`, `js/main.js`로 구성된 Starter 레벨 정적 앱으로 만든다.
- 문장 분리 기능은 작은 JavaScript 함수로 분리해 수동 확인이 쉽도록 한다.
- 문장 성분 분석 결과는 화면 표시가 쉽도록 성분 묶음 배열로 관리한다.
- 교사 수정 기능을 위해 각 성분 묶음에는 고유 ID, 역할, 표시 줄, 순서 값을 둔다.
- 발표 상태는 단순하게 유지한다:
  - 원본 지문
  - 분리된 문장 목록
  - 문장별 성분 분석 결과
  - 교사가 수정한 문장별 성분 배치
  - 현재 문장 번호
  - 표시 설정
- 첫 버전에서는 백엔드, 빌드 도구, 패키지 매니저가 필요하지 않다.
- CSS에서 `Pretendard`를 기본 글씨체로 지정하고, 발표 문장에는 큰 글자 크기와 높은 대비를 우선 적용한다.
- CSS에서 문장 성분 묶음에는 줄바꿈 방지 스타일을 적용한다.

## 13. 규칙 전제

- 파일 이름은 명확하게 짓고 함수는 작게 유지한다.
- HTML은 구조, CSS는 모양, JavaScript는 동작을 담당하게 한다.
- 버튼과 입력 요소는 의미에 맞는 HTML 요소를 사용한다.
- 실제 필요가 생기기 전에는 불필요한 라이브러리를 추가하지 않는다.
- 앞으로 생성하는 PDCA 문서는 한글로 작성한다.
