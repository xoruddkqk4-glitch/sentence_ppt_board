# sentence-presentation-app - 설계 문서

> 버전: 1.0.0 | 작성일: 2026-05-30 | 상태: 완료
> 프로젝트 레벨: Starter | 계획 문서: `docs/01-plan/features/sentence-presentation-app.plan.md`

## 1. 개요

이 앱은 영어 교사가 지문을 붙여넣으면 문장을 자동으로 나누고, 각 문장을 주요소와 종요소로 계층화해 교실 발표 화면에 크게 표시하는 정적 웹 앱이다.

첫 버전은 서버 없이 `index.html`, `css/style.css`, `js/main.js`만으로 실행한다. 자동 문장 성분 분석은 완벽한 정답 생성이 아니라 교사가 수업 전에 빠르게 고칠 수 있는 초안 생성으로 설계한다.

## 2. 핵심 설계 원칙

- 발표 화면의 최우선 목표는 교실 맨 뒤 학생도 읽을 수 있는 가시성이다.
- 전체 UI 글씨체는 `Pretendard`, `system-ui`, `sans-serif` 순서로 적용한다.
- 주요소는 맨 윗줄 한 줄에 모두 크고 굵게 표시한다.
- 발표 화면의 주요소에는 라벨을 붙이지 않고 성분별 색으로 구분한다.
- 주어, 동사, 목적어, 보어의 색상 범례는 발표 화면 위쪽에 표시한다.
- 종요소는 주요소 아래에 성분 하나당 한 줄로 표시하고, 종요소 전용 단일 색을 사용한다.
- 종요소는 문장 안 원래 위치를 기준으로 해당 줄에서 가운데 정렬한다.
- 같은 문장 성분에 속한 단어 묶음은 중간에서 줄바꿈되지 않는다.
- 자동 분석 결과는 교사가 드래그앤드롭으로 수정할 수 있어야 한다.
- 발표 중 실수 방지를 위해 `수정 모드`와 `발표 모드`를 구분한다.

## 3. 페이지 구조

### 3.1 단일 페이지 구성

앱은 하나의 HTML 페이지 안에서 세 화면 상태를 전환한다.

| 화면 상태 | 목적 | 주요 요소 |
|-----------|------|-----------|
| 입력 화면 | 교사가 영어 지문을 입력한다. | 지문 입력창, 샘플 지문 버튼, 발표 시작 버튼 |
| 수정 화면 | 자동 분석 결과를 교사가 확인하고 고친다. | 성분 묶음, 주요소 줄, 종요소 줄, 역할 선택, 드래그 핸들 |
| 발표 화면 | 학생에게 문장을 크게 보여준다. | 색상 범례, 주요소 한 줄, 종요소 개별 줄, 진행 표시, 이전/다음 버튼 |

### 3.2 기본 사용자 흐름

1. 교사가 입력 화면에서 지문을 붙여넣는다.
2. 교사가 `발표 준비` 버튼을 누른다.
3. 앱이 지문을 문장 단위로 나눈다.
4. 앱이 각 문장에 대해 규칙 기반 성분 분석 초안을 만든다.
5. 교사는 수정 화면에서 드래그앤드롭으로 분석 결과를 고친다.
6. 교사가 `발표 시작` 버튼을 누른다.
7. 발표 화면에서 문장을 하나씩 넘기며 수업한다.
8. 필요하면 수정 화면으로 돌아가 성분 배치를 다시 고친다.

## 5. 화면 설계

### 5.1 입력 화면

입력 화면은 수업 준비용 화면이다. 발표용 장식보다 빠른 입력과 실행을 우선한다.

구성:

- 앱 제목
- 지문 입력 `textarea`
- 입력 초기화 버튼
- 샘플 지문 넣기 버튼
- 발표 준비 버튼
- 빈 입력 또는 문장 없음 안내 메시지

동작:

- `발표 준비`를 누르면 문장 분리와 성분 분석을 실행한다.
- 입력값이 비어 있으면 발표 준비를 막고 안내 메시지를 보여준다.
- 기존 입력값은 수정 화면과 발표 화면을 오가도 유지한다.

### 5.2 수정 화면

수정 화면은 교사가 자동 분석 결과를 확인하고 수업에 맞게 고치는 화면이다.

구성:

- 현재 문장 원문
- 주요소 드롭 영역
- 종요소 드롭 영역
- 성분 묶음 카드
- 성분 역할 선택 메뉴
- 이전/다음 문장 버튼
- 발표 시작 버튼
- 입력 화면으로 돌아가기 버튼

성분 묶음 카드 표시:

```text
[주어] The young students
[동사] read
[목적어] the story
[부사어] carefully
```

드래그앤드롭 규칙:

- 성분 묶음 전체를 하나의 이동 단위로 다룬다.
- 성분 묶음 내부 텍스트는 드래그 중에도 분리되지 않는다.
- 같은 줄 안에서는 성분 순서를 바꿀 수 있다.
- 주요소 줄과 종요소 줄 사이로 성분을 이동할 수 있다.
- 줄을 이동하면 성분 역할 선택 메뉴로 역할도 바꿀 수 있다.
- 마우스와 전자칠판 터치를 고려해 큰 드래그 핸들을 둔다.

### 5.3 발표 화면

발표 화면은 학생용 화면이다. 문장 성분과 문장 자체가 가장 크게 보여야 한다.

구성:

- 상단의 작은 진행 표시: `3 / 12`
- 색상 범례: 주어, 동사, 목적어, 보어, 종요소
- 주요소 한 줄: 주어, 동사, 보어, 목적어
- 종요소 줄: 형용사어, 부사어를 성분 하나당 한 줄로 표시
- 하단 조작 버튼: 이전, 다음, 수정, 편집
- 텍스트 길이에 따른 자동 글자 크기 조정
- 밝은 테마 / 어두운 테마 전환

표시 규칙:

- 주요소 줄은 한 줄에 모두 들어가게 하고 가장 큰 글씨와 굵은 글씨를 사용한다.
- 주요소 텍스트에는 성분 라벨을 붙이지 않고 색상으로만 구분한다.
- 종요소 줄은 주요소보다 작지만 교실 뒤에서도 읽을 수 있는 크기를 유지한다.
- 종요소가 1개면 문장 영역은 총 2줄, 종요소가 2개면 총 3줄이 된다.
- 종요소는 문장 안 위치를 기준으로 단어 위치 그리드에 배치한다.
- 동사와 목적어 사이의 부사어는 해당 종요소 줄에서 동사와 목적어 사이를 기준으로 가운데 정렬한다.
- 종요소가 들어가는 위치에는 단어와 단어 사이 기준으로 점을 찍고, 점에서 종요소 네모칸까지 연결선을 표시한다.
- 주요소 단어가 3개면 종요소 삽입 슬롯은 5개, 4개면 6개처럼 주요소 단어 수보다 2개 많은 슬롯을 제공한다.
- 종요소 글자 크기는 주요소보다 작게 유지하되 화면 폭에 따라 동적으로 조정한다.
- Color Hunt 팔레트 `#00B8A9`, `#F8F3D4`, `#F6416C`, `#FFDE7D`는 문장 성분 색에만 사용하고 웹페이지 기본색은 기존 색을 유지한다.
- 발표 화면에서 종요소 네모칸은 드래그앤드롭으로 삽입 위치를 옮길 수 있다.
- 각 성분 묶음은 `white-space: nowrap`으로 내부 줄바꿈을 막는다.
- 줄바꿈이 필요하면 성분 묶음 사이에서만 줄바꿈한다.
- 발표 화면은 카드 안에 갇힌 느낌이 아니라 전체 화면을 넓게 사용한다.

## 7. 문장 처리 설계

### 7.1 문장 분리

첫 버전은 규칙 기반 분리 함수를 사용한다.

처리 순서:

1. 앞뒤 공백을 제거한다.
2. 줄바꿈을 하나의 공백으로 정리한다.
3. 마침표, 물음표, 느낌표 뒤의 공백을 기준으로 문장을 나눈다.
4. 닫는 따옴표가 붙은 문장은 문장 끝으로 유지한다.
5. 빈 문장은 제거한다.

알려진 한계:

- `Mr.`, `Dr.`, `e.g.` 같은 약어는 일부 잘못 분리될 수 있다.
- 이 문제는 첫 버전에서 완벽히 해결하기보다 수동 수정 가능 구조로 보완한다.

### 7.2 성분 분석 초안

첫 버전의 성분 분석은 간단한 규칙 기반 초안으로 만든다.

기본 전략:

- 문장을 단어 단위로 나눈다.
- 자주 쓰이는 be동사와 일반 동사 후보를 기준으로 동사 위치를 찾는다.
- 동사 앞쪽을 주어 후보로 둔다.
- 동사 뒤쪽을 목적어 또는 보어 후보로 둔다.
- 전치사구, 시간/장소 표현, `-ly` 부사는 부사어 후보로 둔다.
- 관사, 소유격, 형용사 후보는 가까운 명사를 꾸미는 형용사어 후보로 둔다.

중요한 전제:

- 자동 분석은 정확한 문법 판정기가 아니라 교사용 초안이다.
- 교사가 수정 화면에서 드래그앤드롭과 역할 변경으로 보정한다.

## 8. 드래그앤드롭 설계

### 8.1 조작 방식

Starter 레벨 구현에서는 외부 라이브러리 없이 브라우저 기본 기능을 우선 검토한다.

권장 방식:

- 데스크톱: HTML Drag and Drop API
- 터치/전자칠판 대응: Pointer Events 기반 보완

첫 구현에서는 마우스 드래그를 먼저 완성하고, 터치 대응은 같은 데이터 구조 위에서 확장한다.

### 8.2 드롭 영역

드롭 영역은 두 개다.

| 드롭 영역 | lane 값 | 허용 성분 |
|-----------|---------|-----------|
| 주요소 줄 | major | 주어, 동사, 보어, 목적어 |
| 종요소 줄 | minor | 형용사어, 부사어 |

드롭 동작:

- 성분을 주요소 줄에 놓으면 `lane`을 `major`로 바꾼다.
- 성분을 종요소 줄에 놓으면 `lane`을 `minor`로 바꾼다.
- 같은 줄 안에서 위치를 바꾸면 `order`를 다시 계산한다.
- 역할이 줄과 맞지 않으면 역할 선택을 유도하거나 기본 역할로 바꾼다.

### 8.3 역할 수정

각 성분 묶음에는 역할 선택 메뉴를 둔다.

선택 가능 값:

- 주어
- 동사
- 보어
- 목적어
- 형용사어
- 부사어

역할을 바꾸면:

- 주요소 역할이면 `lane`을 `major`로 설정한다.
- 종요소 역할이면 `lane`을 `minor`로 설정한다.
- 화면은 즉시 다시 렌더링된다.

### 8.4 실수 방지

- 수정 화면에서는 전체 성분 묶음 드래그앤드롭을 활성화한다.
- 발표 화면에서는 주요소는 움직이지 않고, 종요소 네모칸만 삽입 위치 조정을 위해 움직일 수 있다.
- 드래그 가능한 성분에는 큰 핸들을 표시한다.
- 드롭 가능한 줄은 드래그 중에 테두리나 배경색으로 강조한다.

## 9. 스타일 설계

### 9.1 글씨체

CSS 기본값:

```css
:root {
  font-family: "Pretendard", system-ui, sans-serif;
}
```

웹 폰트 적용:

- 인터넷 연결이 가능하면 Pretendard CDN을 사용할 수 있다.
- 오프라인 실행 가능성을 위해 대체 글꼴을 반드시 둔다.
- 구현 단계에서 로컬 폰트 파일 포함 여부를 검토한다.

### 9.2 발표 화면 크기 기준

기본 방향:

- 주요소: 매우 큰 글씨, 굵은 두께
- 종요소: 주요소보다 작지만 크게 읽히는 글씨
- 진행 표시와 버튼: 발표 문장을 방해하지 않는 작은 크기

예상 CSS 변수:

```css
:root {
  --major-font-size: clamp(56px, 7vw, 124px);
  --minor-font-size: clamp(36px, 4.5vw, 78px);
  --component-gap: clamp(16px, 2vw, 36px);
}
```

### 9.3 성분 묶음 줄바꿈 방지

```css
.component-chip {
  display: inline-flex;
  align-items: baseline;
  white-space: nowrap;
}
```

줄 컨테이너는 성분 묶음 사이에서만 줄바꿈한다.

```css
.component-lane {
  display: flex;
  flex-wrap: wrap;
  justify-content: center;
  gap: var(--component-gap);
}
```

### 9.4 색상

밝은 테마:

- 배경: 거의 흰색
- 주요 텍스트: 검정에 가까운 색
- 주요소 강조: 진한 파란색 또는 검정
- 종요소: 진한 회색

어두운 테마:

- 배경: 검정에 가까운 색
- 주요 텍스트: 흰색
- 주요소 강조: 흰색
- 종요소: 밝은 회색

색상은 한 가지 색 계열에 치우치지 않게 하고, 발표 문장 대비를 최우선으로 둔다.

## 10. 키보드 설계

| 키 | 동작 |
|----|------|
| 오른쪽 화살표 | 다음 문장 |
| Space | 다음 문장 |
| 왼쪽 화살표 | 이전 문장 |
| Home | 첫 문장 |
| End | 마지막 문장 |

## 11. 주요 함수 설계

| 함수 | 역할 |
|------|------|
| `splitSentences(text)` | 지문을 문장 배열로 나눈다. |
| `analyzeSentence(sentenceText)` | 문장 성분 초안을 만든다. |
| `renderInputView()` | 입력 화면을 그린다. |
| `renderEditView()` | 수정 화면을 그린다. |
| `renderPresentView()` | 발표 화면을 그린다. |
| `moveComponent(componentId, targetLane, targetIndex)` | 성분 묶음을 이동한다. |
| `updateComponentRole(componentId, role)` | 성분 역할을 변경한다. |
| `goToSentence(index)` | 현재 문장 번호를 바꾼다. |
| `applySettings(settings)` | 테마와 자동 글자 크기 기준값을 반영한다. |

## 12. 수동 QA 계획

### 12.1 입력 검증

- 빈 입력에서 발표 준비를 누르면 안내 메시지가 보인다.
- 공백과 줄바꿈이 많은 지문도 문장으로 정리된다.
- 짧은 한 문장 지문도 정상 표시된다.

### 12.2 문장 표시

- 주요소가 첫 줄에 굵게 표시된다.
- 종요소가 두 번째 줄에 약간 작은 글씨로 표시된다.
- 같은 성분 묶음 내부에서는 줄바꿈이 일어나지 않는다.
- 화면이 좁아지면 성분 묶음 사이에서만 줄바꿈된다.

### 12.3 교사 수정

- 성분 묶음을 같은 줄 안에서 이동할 수 있다.
- 성분 묶음을 주요소 줄에서 종요소 줄로 옮길 수 있다.
- 성분 역할을 바꾸면 줄과 표시가 즉시 갱신된다.
- 발표 화면으로 갔다가 돌아와도 수정 결과가 유지된다.

### 12.4 발표 조작

- 이전/다음 버튼이 작동한다.
- 방향키와 Space 키가 작동한다.
- 텍스트 길이에 따라 발표 글자 크기가 자동 조정된다.
- 밝은 테마와 어두운 테마 전환이 작동한다.

## 16. Iterate 반영 설계 갱신

> 반영일: 2026-05-30  
> 근거 분석 문서: `docs/03-analysis/sentence-presentation-app.analysis.md`

이번 갱신은 구현 코드에 이미 반영된 최신 수업 흐름을 설계 문서에 맞추기 위한 것이다. 초기 설계는 입력, 수정, 발표의 기본 구조를 중심으로 작성되었고, 이후 수업 사용성 요구가 늘어나면서 저장, 단계별 공개, 형용사어 수식, 세밀한 키보드 조작, 긴 문장 보정이 추가되었다.

### 16.1 최신 화면 흐름

앱은 여전히 단일 정적 웹 페이지로 동작한다.

1. **입력 화면**
   - 교사가 영어 지문을 입력한다.
   - 샘플 지문을 넣을 수 있다.
   - 분석 결과가 있는 경우 TXT로 저장하거나 TXT 파일에서 불러올 수 있다.

2. **분석 결과 수정 화면**
   - 문장별 주요소와 종요소를 확인한다.
   - 성분 역할을 선택 메뉴로 바꿀 수 있다.
   - 드래그 앤 드랍으로 주요소/종요소 lane 사이를 이동할 수 있다.
   - 하나의 구로 묶인 표현 중 교사가 선택한 단어만 분리할 수 있다.
   - 분리된 성분을 서로 겹치게 드래그하면 병합할 수 있다.
   - 형용사어는 수식 대상을 원문 단어 기준으로 복수 선택할 수 있다.

3. **프리젠테이션 화면**
   - 첫 단계에서는 주요소만 보여준다.
   - 다음 단계부터 종요소를 하나씩 누적 공개한다.
   - 종요소가 모두 공개된 뒤 다음 동작을 하면 다음 문장으로 이동한다.
   - 주요소와 종요소는 같은 성분 안에서 줄바꿈되지 않는다.
   - 종요소는 한 성분이 한 줄에 보이도록 표시한다.

### 16.2 저장과 불러오기

분석 결과는 TXT 파일로 저장할 수 있어야 한다. TXT 파일에는 사람이 읽을 수 있는 요약과 앱이 다시 불러올 수 있는 JSON 블록을 함께 둔다.

저장 데이터에는 다음 항목을 포함한다.

- 원문 지문
- 문장 목록
- 문장 인덱스
- 성분 ID
- 성분 텍스트
- 성분 역할
- 주요소/종요소 lane
- 원문 단어 기준 시작/끝 인덱스
- 형용사어 수식 대상 인덱스
- 화면 설정

브라우저가 `showSaveFilePicker`를 지원하면 저장 위치 선택 창을 사용한다. 지원하지 않는 브라우저에서는 일반 다운로드 방식으로 저장한다.

### 16.3 프리젠테이션 조작

프리젠테이션 화면의 기본 조작은 다음과 같다.

| 조작 | 동작 |
|------|------|
| 오른쪽 방향키 | 다음 단계 |
| Space | 다음 단계 |
| 화면 클릭 | 다음 단계 |
| 왼쪽 방향키 | 이전 단계 |
| 아래 방향키 | 다음 문장 첫 단계 |
| 위 방향키 | 이전 문장 첫 단계 |
| Home | 첫 문장 |
| End | 마지막 문장 마지막 단계 |
| Esc | 수정 화면으로 이동 |

화면 하단에는 항상 네 개의 큰 버튼을 둔다.

```text
첫 문장 | 이전 | 다음 | 마지막 문장
```

마지막 문장에서는 네 번째 버튼의 텍스트를 `이전 단계`로 바꾸고, 해당 버튼은 실제 이전 단계로 이동한다.

### 16.4 종요소 표시 정책

종요소는 주요소 줄 아래에서 누적 공개한다.

- 종요소는 개수 제한 없이 생성할 수 있다.
- 종요소 하나는 한 줄에 표시한다.
- 같은 문장에서 공개된 종요소는 누적해서 유지한다.
- 종요소가 들어갈 수 있는 위치는 문장의 맨 앞, 주요소 단어 사이, 문장의 맨 뒤다.
- 종요소 위치를 나타내는 점은 주요소 단어 위나 단어 내부에 겹치면 안 된다.
- 같은 위치나 가까운 위치에 여러 종요소가 있으면 점과 연결선을 좌우로 분산한다.
- 종요소가 화면 끝에 올 때는 종요소 박스 폭과 연결점 공간까지 고려해 주요소 글자 크기를 조정한다.
- 긴 종요소는 가로 스크롤 없이 화면 안에 들어오도록 종요소 글자 크기를 동적으로 줄인다.

### 16.5 형용사어 수식 표시

형용사어는 일반 부사구 종요소와 다르게 수식 대상을 보여준다.

- 수정 화면에서 교사는 형용사어가 수식하는 원문 단어를 복수 선택할 수 있다.
- 수식 대상은 주요소 단어뿐 아니라 이미 공개된 종요소 내부 단어도 될 수 있다.
- 프리젠테이션 화면에서는 수식 대상 단어 묶음을 하나의 괄호로 감싼다.
- 형용사어 박스에서 수식 대상 괄호 아래까지 화살표를 연결한다.
- 화살표 머리는 단어와 겹치지 않게 괄호 아래에 둔다.

### 16.6 구두점과 apostrophe 보존

문장 분석과 프리젠테이션 표시에서 구두점을 보존한다.

- 마침표, 쉼표, 물음표, 느낌표, 세미콜론, 콜론은 가까운 단어에 붙여 표시한다.
- `Let's`, `today's` 같은 apostrophe 포함 단어는 한 단어로 유지한다.
- `’`, `` ` ``, `´`, `ʼ`처럼 apostrophe처럼 입력되는 문자는 표준 `'`로 정규화한다.
- 이미 `Let s`처럼 갈라진 흔한 축약형은 표시 단계에서 `Let's`로 다시 묶는다.

### 16.7 설정 정책 변경

초기 설계의 글자 크기 수동 조절과 정렬 설정은 최신 요구사항에서 우선순위가 낮아졌다. 현재 설계 기준은 다음과 같다.

- 글자 크기는 텍스트 길이와 화면 폭에 따라 자동 조정한다.
- 별도 글자 크기 메뉴는 제공하지 않는다.
- 발표 화면의 문장 배치는 중앙 정렬을 기본으로 한다.
- `state.settings.align` 같은 내부 확장 필드는 남을 수 있으나, 현재 사용자 UI 요구사항은 아니다.
- 기본 테마 전환은 유지한다.

### 16.8 시각 QA 기준

다음 예시는 iterate 이후 반드시 확인해야 하는 시각 QA 항목이다.

- 긴 주요소가 한 줄에 들어오는가.
- 긴 종요소가 화면 밖으로 나가지 않는가.
- 종요소가 마지막 단어 뒤에 올 때 점이 단어와 겹치지 않는가.
- 같은 위치에 종요소가 여러 개 있을 때 점과 선이 겹치지 않는가.
- 형용사어 괄호가 수식 대상 전체를 하나로 감싸는가.
- 형용사어 화살표가 단어와 겹치지 않는가.
- `Let's`, `today's` 같은 apostrophe가 프리젠테이션 화면에 보이는가.
- 저장한 TXT 파일을 불러오면 문장, 성분, 수식 대상, 인덱스가 복원되는가.
