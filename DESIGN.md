# whiteboard_ppt 디자인 가이드

> 기준 기능: `sentence-presentation-app`
> 프로젝트 레벨: Starter
> 기준 문서: `docs/01-plan/features/sentence-presentation-app.plan.md`, `docs/02-design/features/sentence-presentation-app.design.md`

---

## 1. 디자인 방향

`whiteboard_ppt`는 영어 교사가 지문을 문장 단위로 발표하고, 문장 성분을 수업 중 설명할 수 있게 돕는 교실용 화면 앱이다.

이 디자인의 최우선 목표는 예쁜 장식이 아니라 교실 맨 뒤 학생도 읽을 수 있는 가시성이다. 발표 화면에서는 문장과 문장 성분이 화면의 주인공이어야 하며, UI 버튼과 설정 도구는 수업 흐름을 방해하지 않아야 한다.

기존 참고 문서에서 남길 원칙:

- UI는 조용하고 절제되게 만든다.
- 화면 전환과 색상 대비로 구조를 만든다.
- 불필요한 그림자, 장식, 그라디언트를 넣지 않는다.
- 버튼과 조작 요소는 명확한 색과 충분한 터치 영역을 가진다.
- 8px 기반 간격 체계를 사용한다.
- 반응형 화면에서도 텍스트가 겹치거나 잘리지 않게 한다.

기존 참고 문서에서 제거한 방향:

- 제품 사진 중심의 쇼핑 페이지 구성
- Apple 제품 타일, 스토어 카드, iPhone 구성기 패턴
- SF Pro 전용 타이포그래피
- 제품 이미지용 그림자와 사진 중심 레이아웃

## 2. 핵심 UX 원칙

### 2.1 학생 화면 우선

- 발표 화면은 학생이 보는 화면이다.
- 교실 뒤쪽에서도 읽히도록 글자 크기, 대비, 줄 간격을 최우선으로 둔다.
- 발표 문장을 작은 카드 안에 넣지 않고 화면 전체를 넓게 사용한다.
- 화면에 보이는 설명 문구는 최소화한다.

### 2.2 교사 조작은 빠르게

- 교사는 수업 중 마우스, 키보드, 전자칠판 터치로 빠르게 조작할 수 있어야 한다.
- 주요 작업은 큰 버튼으로 제공한다.
- 수정 화면과 발표 화면을 분리해 발표 중 실수로 문장 성분이 움직이지 않게 한다.
- 드래그앤드롭 대상은 손가락으로도 잡기 쉽게 충분히 커야 한다.

### 2.3 자동 분석은 초안

- 문장 성분 자동 분석은 틀릴 수 있다.
- 분석 결과는 교사가 수정할 수 있는 초안으로 보여준다.
- 드래그앤드롭과 역할 수정이 자연스럽게 이어져야 한다.

## 3. 화면 구조

앱은 단일 페이지에서 세 가지 화면 상태를 전환한다.

| 화면 | 사용자 | 목적 |
|------|--------|------|
| 입력 화면 | 교사 | 영어 지문을 붙여넣고 발표 준비를 시작한다. |
| 수정 화면 | 교사 | 자동 분석된 문장 성분을 드래그앤드롭으로 보정한다. |
| 발표 화면 | 학생, 교사 | 주요소 한 줄과 종요소 개별 줄을 색상 기준으로 크게 보여준다. |

## 4. 색상

색상은 기능 중심으로 제한한다. 장식용 다색 팔레트보다 높은 대비와 명확한 조작 상태를 우선한다.

### 4.1 토큰

| 토큰 | 값 | 용도 |
|------|----|------|
| `--color-bg` | `#ffffff` | 밝은 테마 기본 배경 |
| `--color-bg-soft` | `#f6f7f9` | 입력/수정 화면 보조 배경 |
| `--color-ink` | `#101114` | 밝은 테마 주요 텍스트 |
| `--color-muted` | `#5d6675` | 보조 텍스트 |
| `--color-primary` | `#0066cc` | 주요 버튼, 포커스, 선택 상태 |
| `--color-primary-strong` | `#004f9e` | 버튼 누름 상태 |
| `--color-border` | `#d9dee7` | 입력창, 드롭 영역 경계 |
| `--color-major` | `#101114` | 주요소 텍스트 |
| `--color-minor` | `#334155` | 종요소 텍스트 |
| `--color-drop` | `#e8f2ff` | 드래그 중 드롭 가능 영역 |
| `--color-dark-bg` | `#050608` | 어두운 테마 배경 |
| `--color-dark-ink` | `#f8fafc` | 어두운 테마 주요 텍스트 |
| `--color-dark-muted` | `#cbd5e1` | 어두운 테마 보조 텍스트 |

### 4.2 사용 규칙

- 조작 가능한 요소는 `--color-primary`를 중심으로 통일한다.
- 발표 화면은 밝은 테마와 어두운 테마 모두 강한 대비를 유지한다.
- 문장 성분을 꾸미기 위해 여러 색을 남발하지 않는다.
- 색상만으로 성분을 구분하지 말고 위치, 굵기, 크기, 라벨을 함께 사용한다.
- 장식용 그라디언트는 사용하지 않는다.

## 5. 타이포그래피

### 5.1 글꼴

전체 UI와 발표 화면은 `Pretendard`를 기본으로 사용한다.

```css
:root {
  font-family: "Pretendard", system-ui, sans-serif;
}
```

규칙:

- `Pretendard` 로딩 실패에 대비해 `system-ui`, `sans-serif` 대체 글꼴을 둔다.
- 글자 간격은 기본값 `0`을 유지한다.
- 발표 화면에서 음수 letter-spacing을 쓰지 않는다.
- 문장 성분이 읽히는 것이 우선이므로 과한 장식 폰트는 사용하지 않는다.

### 5.2 크기 체계

| 토큰 | 값 | 용도 |
|------|----|------|
| `--font-ui-sm` | `14px` | 작은 보조 정보 |
| `--font-ui` | `16px` | 기본 UI |
| `--font-ui-lg` | `20px` | 주요 버튼, 설정 라벨 |
| `--font-major` | `clamp(56px, 7vw, 124px)` | 발표 화면 주요소 |
| `--font-minor` | `clamp(36px, 4.5vw, 78px)` | 발표 화면 종요소 |
| `--font-role` | `clamp(16px, 1.4vw, 24px)` | 성분 라벨 |

### 5.3 굵기와 줄 간격

- 주요소: `font-weight: 800`, `line-height: 1.12`
- 종요소: `font-weight: 650`, `line-height: 1.2`
- UI 본문: `font-weight: 400`, `line-height: 1.5`
- 버튼: `font-weight: 700`

주요소는 가장 먼저 눈에 들어와야 하므로 굵고 크게 쓴다. 종요소는 한 단계 낮은 위계로 보이되, 교실 뒤에서도 읽힐 만큼 충분히 크게 유지한다.

## 6. 레이아웃

### 6.1 간격 체계

8px 기반 간격을 사용한다.

| 토큰 | 값 | 용도 |
|------|----|------|
| `--space-1` | `4px` | 아주 작은 간격 |
| `--space-2` | `8px` | 기본 간격 |
| `--space-3` | `12px` | 조밀한 UI 간격 |
| `--space-4` | `16px` | 버튼 내부 간격 |
| `--space-6` | `24px` | 패널 내부 간격 |
| `--space-8` | `32px` | 화면 블록 간격 |
| `--space-12` | `48px` | 발표 줄 간격 |
| `--space-16` | `64px` | 큰 화면 여백 |

### 6.2 발표 화면

발표 화면은 전체 뷰포트를 사용한다.

구조:

```text
┌────────────────────────────────────┐
│ 3 / 12                  설정/수정  │
│                                    │
│ 범례: 주어 동사 목적어 보어 종요소 │
│                                    │
│ The young students  read  the story │
│                         carefully  │
│                                    │
│ 이전                         다음  │
└────────────────────────────────────┘
```

규칙:

- 문장 성분 영역은 화면 중앙에 둔다.
- 상단 진행 표시와 하단 버튼은 문장을 방해하지 않게 작게 둔다.
- 주요소는 한 줄에 모두 들어가게 한다.
- 주요소에는 성분 라벨을 붙이지 않고 밝은 파스텔톤 색상으로 성분을 구분한다.
- 주요소는 실제 한 줄 너비를 측정해 화면에 맞도록 글자 크기를 동적으로 줄인다.
- 색상 범례는 발표 화면 위쪽에 표시한다.
- 종요소는 하나의 성분이 한 줄씩 차지하고 주요소보다 어두운 톤으로 표시한다.
- 종요소가 들어가는 문장 위치에는 주요소 단어 사이 기준으로 점을 찍고, 점은 주요소 단어 가운데 놓이지 않아야 한다.
- 주요소 단어가 3개면 종요소 삽입 슬롯은 5개, 4개면 6개처럼 주요소 단어 수보다 2개 많은 슬롯을 제공한다.
- 종요소 점은 한 세로 위치에 고정하고, 각 종요소 네모칸까지 연결선을 내린다.
- 두 번째 종요소 줄의 연결선은 첫 번째 종요소 줄의 연결선보다 길어야 한다.
- 종요소 글자 크기도 한 줄 너비를 기준으로 동적으로 줄이며, 주요소보다 작게 유지한다.
- 종요소가 문장 끝에 오더라도 종요소 네모칸 폭까지 고려해 전체 발표 영역이 가로 스크롤 없이 보여야 한다.
- Color Hunt 팔레트 `#00B8A9`, `#F8F3D4`, `#F6416C`, `#FFDE7D`는 문장 성분 색에만 사용하고, 웹페이지 기본 배경과 버튼 색은 기존 기본색을 유지한다.
- 발표 화면에서는 종요소 네모칸을 드래그앤드롭해 문장 안 삽입 위치를 조정할 수 있다.
- 종요소 줄은 문장 안 위치를 기준으로 가로 위치를 맞춘다.
- 동사와 목적어 사이에 있는 부사어는 해당 종요소 줄에서 동사와 목적어 사이 기준으로 가운데 정렬한다.
- 발표 텍스트는 화면 너비를 넓게 사용하되, 좌우 끝에 붙지 않게 한다.

### 6.3 수정 화면

수정 화면은 교사 작업용이다.

구조:

```text
원문 문장

주요소
[주어 ...] [동사 ...] [목적어 ...]

종요소
[형용사어 ...] [부사어 ...]

역할 수정 / 이전 / 다음 / 발표 시작
```

규칙:

- 드롭 영역은 명확한 경계와 충분한 높이를 가진다.
- 드래그 중인 성분과 드롭 가능한 영역은 시각적으로 강조한다.
- 성분 카드 안에 드래그 핸들과 역할 선택 컨트롤을 둔다.
- 수정 화면에서는 정보 밀도가 높아도 되지만, 터치 대상은 작아지면 안 된다.

## 7. 문장 성분 컴포넌트

### 7.1 성분 묶음

성분 묶음은 텍스트, 역할 라벨, 드래그 핸들을 포함한다.

```text
[⋮⋮] [주어] The young students
```

필수 규칙:

- 하나의 성분 묶음은 하나의 시각적 단위다.
- 성분 묶음 내부 단어는 줄바꿈되지 않는다.
- 줄바꿈은 성분 묶음 사이에서만 일어난다.
- 발표 화면에서는 드래그 핸들을 숨긴다.
- 수정 화면에서는 드래그 핸들을 크게 보여준다.

기본 CSS 방향:

```css
.component-lane {
  display: flex;
  flex-wrap: wrap;
  justify-content: center;
  align-items: center;
  gap: var(--component-gap);
}

.component-chip {
  display: inline-flex;
  align-items: baseline;
  max-width: 100%;
  white-space: nowrap;
}
```

### 7.2 주요소와 종요소

주요소:

- 대상: 주어, 동사, 보어, 목적어
- 위치: 맨 윗줄 한 줄
- 스타일: 가장 큰 글자, 진한 굵기, 성분별 색상
- 라벨: 발표 화면에서는 숨기고 위쪽 범례로만 안내

종요소:

- 대상: 형용사어, 부사어
- 위치: 주요소 아래, 성분 하나당 한 줄
- 스타일: 주요소보다 약간 작은 글자, 종요소 전용 단일 색상
- 정렬: 문장 안 원래 위치를 기준으로 같은 단어 위치 그리드에 배치

## 8. 버튼과 컨트롤

### 8.1 버튼 원칙

- 모든 주요 버튼은 최소 44px 높이를 가진다.
- 전자칠판과 태블릿 터치를 고려해 실제 터치 영역은 48px 이상을 권장한다.
- 주요 버튼은 `--color-primary` 배경을 사용한다.
- 보조 버튼은 흰 배경, 진한 텍스트, 얇은 경계를 사용한다.
- 활성 상태는 색상 변화와 작은 scale 변화로 표현할 수 있다.

```css
.button:active {
  transform: scale(0.97);
}

.button:focus-visible {
  outline: 3px solid var(--color-primary);
  outline-offset: 3px;
}
```

### 8.2 버튼 종류

| 컴포넌트 | 용도 |
|----------|------|
| `button-primary` | 발표 준비, 발표 시작, 다음 |
| `button-secondary` | 이전, 편집, 수정 화면으로 |
| `button-icon` | 설정, 닫기, 전체화면 등 보조 조작 |
| `role-select` | 성분 역할 변경 |
| `font-scale-control` | 발표 글자 크기 조절 |
| `theme-toggle` | 밝은 테마와 어두운 테마 전환 |

## 9. 드래그앤드롭 상태

| 상태 | 표시 |
|------|------|
| 기본 | 성분 묶음이 안정적으로 보인다. |
| 드래그 가능 | 핸들이 보이고 커서가 이동 가능 상태다. |
| 드래그 중 | 성분 묶음이 살짝 떠 보이고 투명도가 낮아진다. |
| 드롭 가능 | 주요소/종요소 줄 배경이 `--color-drop`으로 강조된다. |
| 드롭 완료 | 새 위치에 즉시 정렬되고 역할이 갱신된다. |

주의:

- 발표 화면에서는 주요소 드래그 상태가 없어야 한다.
- 발표 화면의 종요소 네모칸은 삽입 위치 조정용 드래그 상태를 가질 수 있다.
- 드래그 중에도 성분 내부 줄바꿈 금지 규칙을 유지한다.
- 터치 환경에서 실수로 이동하지 않도록 수정 화면에서만 드래그를 허용한다.

## 10. 반응형 기준

| 이름 | 너비 | 주요 변화 |
|------|------|-----------|
| 작은 휴대폰 | `<= 419px` | UI는 세로 스택, 발표 글자는 최소값 사용 |
| 휴대폰 | `420px - 640px` | 입력/수정 화면 단일 열 |
| 태블릿 | `641px - 1023px` | 터치 조작 중심, 버튼 간격 확대 |
| 데스크톱 | `1024px - 1440px` | 기본 수업 준비 화면 |
| 대형 교실 화면 | `>= 1441px` | 발표 글자 최대 크기, 넓은 여백 |

반응형 규칙:

- 발표 글자 크기는 `clamp()`로 제어한다.
- 화면이 좁아도 성분 묶음 내부를 강제로 줄바꿈하지 않는다.
- 성분 묶음이 너무 길면 묶음 사이 재배치와 글자 크기 조절을 우선한다.
- 버튼과 드래그 핸들은 작은 화면에서도 44px 미만으로 줄이지 않는다.

## 11. 접근성

- 버튼에는 명확한 `aria-label` 또는 보이는 텍스트를 둔다.
- 드래그앤드롭 기능은 마우스만 전제로 하지 않는다.
- 역할 변경은 `select` 같은 표준 폼 컨트롤을 우선한다.
- 키보드 조작을 지원한다.

키보드:

| 키 | 동작 |
|----|------|
| 오른쪽 화살표 | 다음 문장 |
| Space | 다음 문장 |
| 왼쪽 화살표 | 이전 문장 |
| Home | 첫 문장 |
| End | 마지막 문장 |
| Esc | 발표 화면에서 수정 화면으로 이동 |

입력창에 포커스가 있을 때는 텍스트 입력을 방해하지 않는다.

## 12. 하지 말아야 할 것

- 발표 문장을 작은 카드 안에 가두지 않는다.
- 발표 화면을 장식 이미지나 배경 효과로 복잡하게 만들지 않는다.
- 성분 묶음 내부에서 단어가 줄바꿈되게 만들지 않는다.
- 학생용 발표 화면에서 주요소 드래그 조작이 가능하게 두지 않는다.
- 종요소 위치 조정 드래그는 종요소 네모칸에만 허용한다.
- 글자 크기를 뷰포트 비율만으로 무제한 키우거나 줄이지 않는다.
- 음수 letter-spacing을 사용하지 않는다.
- 한 가지 색 계열만 반복해 화면 전체가 단조롭게 보이게 하지 않는다.
- 버튼과 조작 요소를 44px보다 작게 만들지 않는다.

## 13. 구현 체크리스트

- [ ] `Pretendard` 폰트 스택이 적용되어 있다.
- [ ] 주요소는 윗줄에 크고 굵게 표시된다.
- [ ] 주요소는 한 줄에 모두 들어간다.
- [ ] 발표 화면 주요소에는 라벨이 없고 색상 범례가 화면 위쪽에 표시된다.
- [ ] 종요소는 성분 하나당 한 줄로 표시된다.
- [ ] 종요소는 문장 안 위치를 기준으로 정렬된다.
- [ ] 성분 묶음 내부에서 줄바꿈이 일어나지 않는다.
- [ ] 성분 묶음 사이에서만 줄바꿈된다.
- [ ] 수정 화면에서 드래그앤드롭이 가능하다.
- [ ] 발표 화면에서는 주요소 드래그앤드롭이 비활성화된다.
- [ ] 발표 화면에서는 종요소 네모칸을 드래그해 삽입 위치를 조정할 수 있다.
- [ ] 주요 버튼과 드래그 핸들은 터치하기 충분히 크다.
- [ ] 밝은 테마와 어두운 테마 모두 대비가 충분하다.
- [ ] 휴대폰, 태블릿, 데스크톱, 대형 화면에서 텍스트가 겹치지 않는다.

## 15. 최신 구현 반영

> 반영일: 2026-05-30  
> 관련 기능: `sentence-presentation-app`

현재 앱은 초기 설계보다 더 많은 수업용 기능을 포함한다. 아래 항목은 실제 구현과 설계 문서의 차이를 줄이기 위해 최신 동작 기준으로 추가한다.

### 15.1 핵심 사용 흐름

1. 교사가 영어 지문을 입력한다.
2. 앱이 문장 단위로 분리하고 주어, 동사, 목적어, 보어, 중요소를 자동 분석한다.
3. 교사는 2단계 분석 결과 수정 화면에서 성분을 수정하고, 필요한 경우 구를 단어 단위 또는 선택 단어 묶음 단위로 분리한다.
4. 겹치게 드래그한 성분은 다시 하나의 성분으로 합칠 수 있다.
5. 발표 화면에서는 주요소를 먼저 보여주고, 클릭 또는 다음 동작마다 종요소를 하나씩 누적 공개한다.

### 15.2 발표 화면 내비게이션

발표 화면 하단에는 모든 문장에서 네 개의 이동 버튼을 둔다.

| 버튼 | 동작 |
|------|------|
| 첫 문장 | 현재 발표를 첫 문장으로 이동 |
| 이전 | 같은 문장 안에서는 이전 공개 단계로 이동, 첫 단계에서는 이전 문장으로 이동 |
| 다음 | 같은 문장 안에서는 다음 종요소를 공개, 더 공개할 종요소가 없으면 다음 문장으로 이동 |
| 마지막 문장 / 이전 단계 | 마지막 문장이 아니면 마지막 문장으로 이동, 마지막 문장에서는 이전 단계 화면으로 이동 |

키보드 조작은 다음 기준을 따른다.

| 입력 | 동작 |
|------|------|
| 오른쪽 화살표 | 다음 버튼과 동일 |
| 왼쪽 화살표 | 이전 버튼과 동일 |
| 아래 화살표 | 다음 문장으로 이동 |
| 위 화살표 | 이전 문장으로 이동 |
| 화면 클릭 | 다음 버튼과 동일 |

### 15.3 종요소 공개와 배치

- 발표 첫 화면은 주요소만 표시한다.
- 종요소는 한 번에 하나씩 추가로 공개하고, 이미 공개된 종요소는 계속 누적해서 보여준다.
- 종요소 하나는 한 줄에 하나만 표시한다.
- 종요소 수에는 고정된 최대 개수 제한을 두지 않는다.
- 종요소끼리 겹치지 않도록 줄 위치와 연결선을 순차적으로 보정한다.
- 종요소가 문장 맨 앞이나 맨 뒤에 붙는 경우에도 화면 밖으로 나가지 않아야 한다.
- 동그라미 anchor는 주요소 단어 위가 아니라 문장 맨 앞, 단어와 단어 사이, 문장 맨 뒤 위치에만 놓는다.

### 15.4 형용사어 수식 표시

- 형용사어는 2단계 분석 결과 수정 화면에서 수식 대상을 원본 문장 기준으로 복수 선택할 수 있다.
- 수식 대상이 여러 단어이면 단어별 괄호가 아니라 하나의 묶음 괄호로 표시한다.
- 발표 화면에서는 형용사어 박스에서 수식 대상 괄호까지 화살표를 연결한다.
- 화살표 머리는 글자와 겹치지 않도록 수식 대상 단어 아래쪽까지만 접근한다.
- 형용사어의 수식 대상은 주요소뿐 아니라 이미 표시된 종요소 안의 표현도 될 수 있다.

### 15.5 구두점과 apostrophe 보존

- 원문에 있는 마침표, 쉼표, apostrophe는 분석 결과 수정 화면과 발표 화면 모두에서 보존한다.
- `Let's`, `today's` 같은 축약형은 화면 표시 단계에서 apostrophe가 사라지지 않아야 한다.
- 내부 토큰 처리를 위해 apostrophe를 정규화하더라도 사용자에게 보이는 문장은 원문 표기를 유지한다.

### 15.6 분석 결과 저장과 불러오기

- 교사는 분석된 문장과 성분 인덱스를 txt 파일로 저장할 수 있다.
- 저장 시 가능한 브라우저에서는 저장 위치 선택 창을 사용한다.
- 저장한 txt 파일을 다시 불러오면 문장, 성분, 분리/병합 결과, 종요소 위치, 형용사어 수식 대상이 복원되어야 한다.
- 브라우저가 저장 위치 선택 API를 지원하지 않으면 일반 다운로드 방식으로 대체한다.

### 15.7 설정 정책 변경

- 첫 화면의 수동 글자 크기 메뉴는 제거한다. 발표 글자 크기는 문장 길이와 종요소 폭을 기준으로 자동 조정한다.
- 정렬 설정 UI는 현재 주요 요구사항이 아니므로 별도 노출하지 않는다.
- 발표 화면 끄기 버튼은 하단 네 버튼 정책과 충돌하지 않도록 마지막 문장 화면의 `이전 단계` 흐름으로 대체한다.

### 15.8 시각 QA 체크리스트

- 긴 주요소가 한 줄 안에 들어가는지 확인한다.
- 긴 종요소가 화면 밖으로 나가지 않는지 확인한다.
- 문장 맨 뒤 종요소의 anchor가 마지막 단어와 겹치지 않는지 확인한다.
- apostrophe가 포함된 문장이 발표 화면에서 그대로 보이는지 확인한다.
- 종요소가 여러 개일 때 동그라미와 연결선이 겹치지 않는지 확인한다.
- 형용사어 괄호가 여러 단어를 하나의 묶음으로 감싸는지 확인한다.
