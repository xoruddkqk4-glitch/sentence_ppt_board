# Sentence PPT Board

영어 수업용 문장 프리젠테이션 보드입니다. 교사가 영어 지문을 입력하면 앱이 문장을 나누고, 각 문장을 주어, 동사, 목적어, 보어, 형용사어, 부사어로 분석해 교실 화면에 크게 보여줍니다.

최종 버전은 교실 맨 뒤 학생도 읽을 수 있도록 `Pretendard` 글씨체와 큰 발표 화면을 사용합니다. 주요소는 한 줄에 크게 표시하고, 종요소는 교사가 설명 순서에 맞춰 하나씩 공개할 수 있습니다.

## 최종 버전 핵심 기능

- 영어 지문을 문장 단위로 자동 분리
- 주어, 동사, 목적어, 보어, 제목, 소제목을 주요소(또는 이에 상응하는 주요 라인 성분)로 설정 가능
- 형용사어, 부사어를 종요소로 분석
- 주요소는 라벨 없이 색상으로 구분하고 화면 상단에 범례 표시 (제목/소제목은 발표 화면에 맞춤형 스타일로 노출)
- 종요소가 없는 순수 제목/소제목 문장은 발표 모드에서 중앙 정렬된 큰 타이틀 형식의 단독 블록으로 렌더링
- 제목/소제목 문장 내에 종요소(형용사어, 부사어)가 부여된 경우, 제목 형식을 유지하면서 하단에 종요소 연결선(부사어) 및 수식 괄호/밑줄/화살표(형용사어)가 제목 내 단어들과 이어서 매끄럽게 결합되어 표현됨
- 주요소는 한 줄에 들어오도록 글자 크기를 자동 조정
- 종요소는 한 성분이 한 줄에 표시되도록 배치
- 발표 화면 첫 단계에서는 주요소만 표시
- 클릭, 다음 버튼, 오른쪽 화살표로 종요소를 하나씩 누적 공개
- 종요소가 들어가는 위치를 문장 맨 앞, 단어 사이, 문장 맨 뒤 anchor(동그라미 점)로 표시
- 동그라미 점이 첫/마지막 단어와 겹치지 않도록 주요소 줄의 좌우 패딩을 종요소 줄보다 넓게 확보
- 여러 종요소의 점이 같은 슬롯에 겹쳐서 표시되더라도 발표 카드의 수직 나열 순서에 따라 어순 순서가 시각화되도록 설계 단순화
- 발표 화면을 전체화면으로 전환(오른쪽 위 버튼), 전체화면에서도 배경 유지로 전자칠판 판서 가능
- 종요소 앵커(점/화살표)가 항상 박스의 가로 폭 영역 내에 위치하도록 박스 Clamping 범위를 제한하고, 박스 상단에서 앵커까지 깔끔한 수직 연결선으로 연결
- 종요소 위치를 드래그 앤 드랍으로 각각 독립적으로 이동
- 긴 종요소가 화면 밖으로 나가지 않도록 동적 크기와 위치 보정
- 성분 요소(칩) 클릭 시 해당 요소만 전체 화면에 크게 보여주는 확대 화면(오버레이) 제공 (배경색은 발표 화면 테마와 완벽 동기화)
- 확대 화면 내 텍스트는 1줄에 최대 10단어씩 분할하여 왼쪽 정렬 배치
- 확대 화면 내 줄간격을 기존(2.7) 대비 2배 확장(line-height: 5.4)하여 판서 및 가시성 극대화
- 확대 화면 내 텍스트가 화면 가로폭을 최대한 꽉 채우도록 양방향(확대/축소) 동적 글자 크기 피팅(가로 120px 여백, 세로 200px 여백 및 98% 너비 기준) 적용
- 형용사어가 수식하는 원문 단어 또는 구를 선택 가능(자기 자신의 단어는 대상에서 제외)
- 형용사어는 발표 화면에서 밑줄과 화살표로 수식 관계 표시
- 형용사어 화살표는 수식 대상(주요소 포함)을 나타내는 괄호(bracket)의 밑변 정가운데를 정확히 가리킴
- 주요소와 종요소 안의 apostrophe와 구두점 보존
- 교사가 분석 결과를 드래그 앤 드랍으로 수정
- 선택한 단어만 분리하거나, 선택한 여러 단어를 하나의 묶음으로 분리
- 다른 성분 위에 겹치게 드래그하면 별도 키 없이 병합(병합 텍스트는 문장 순서로 합쳐짐)
- 분석 결과를 인덱스 포함 TXT 파일로 저장
- 저장한 TXT 파일을 다시 불러와 문장과 분석 결과 복원
- "TXT 불러오기" 및 "다른 이름으로 저장" 시 브라우저 표준 파일 피커를 연동하여 최근 작업하던 폴더 위치 기억 및 파일 핸들 갱신 기능 탑재
- "다른 이름으로 저장" 후 "현재 TXT에 저장"을 클릭하면 직전에 새로 저장한 파일에 정확히 덮어쓰기 연동
- 중요 문장 지정(2단계) 및 발표 화면 내 중요도 즉석 토글(3단계 미니 별표 ☆/★) 지원
- 중요 문장들의 순차 단축키(`1`~`9`) 매핑 및 문장 이동용 사이드바 단추 탑재
- 사이드바 내 문장 번호가 단축키 배지 유무에 무관하게 항상 수평 중심에 오도록 대칭형 정렬 구조 적용
- 중요 문서 단축키는 지문 내 등장 순서대로 이가 빠짐없이 압축 및 동적 재정렬 처리
- 2단계 수정 화면에서 1단계 대시보드로 돌아갈 때 텍스트 수정 사항이 1단계 지문에 자동으로 반영 및 취합되도록 개선
- 2단계 수정 화면에서 종요소 칩의 전구(💡) 버튼을 눌러 발표 화면(3단계)에서의 순차 공개 우선순위를 수동 지정 가능 (💡 1, 💡 2, 💡 3...)
- 중간 순위 해제 시 뒷순위 핀 번호 자동 당김(순차 압축) 처리
- 2단계에서는 원문 순서 그대로 칩을 보여주어 읽기 흐름을 방해하지 않고, 3단계 발표 모드에서만 지정된 우선순위대로 정렬하여 순차 공개

## 사용 방법

1. `index.html`을 브라우저에서 엽니다.
2. 입력 화면에 영어 지문을 붙여넣습니다.
3. `발표 준비`를 눌러 문장 분석 결과를 만듭니다.
4. 분석 결과 수정 화면에서 성분 역할, 위치, 분리, 병합, 형용사어 수식 대상을 조정합니다.
5. `발표 시작`을 눌러 학생용 발표 화면으로 전환합니다.
6. 화면 클릭, 버튼, 키보드로 문장과 공개 단계를 진행합니다.

서버나 빌드 도구 없이 동작하는 정적 웹앱입니다.

## 발표 화면 조작

발표 화면 오른쪽 위 `전체화면 (🔍)` 버튼을 누르면 발표 화면이 전체화면으로 전환되고, 다시 누르면 전체화면을 종료합니다. 전체화면에서도 배경은 평소와 동일하게 유지되어 전자칠판 판서에 지장이 없습니다.

하단 버튼은 모든 발표 문장에 표시됩니다.

| 버튼 | 동작 |
|------|------|
| 첫 문장 | 첫 번째 문장으로 이동 |
| 이전 | 같은 문장 안에서는 이전 공개 단계로 이동, 첫 단계에서는 이전 문장으로 이동 |
| 다음 | 같은 문장 안에서는 다음 종요소 공개, 더 공개할 종요소가 없으면 다음 문장으로 이동 |
| 마지막 문장 / 이전 단계 | 마지막 문장이 아니면 마지막 문장으로 이동, 마지막 문장에서는 2단계(분석 결과 수정) 화면으로 이동 |

키보드와 클릭 조작은 다음과 같습니다.

| 입력 | 동작 |
|------|------|
| 1 ~ 9 | 중요 문장 단축키 슬롯 1~9에 해당하는 문장으로 즉시 점프 이동 |
| f 또는 F | 발표화면 전체화면 켜기/끄기 토글 |
| 화면 클릭 | 다음 버튼과 동일 |
| 오른쪽 화살표 | 다음 버튼과 동일 |
| 왼쪽 화살표 | 이전 버튼과 동일 |
| 아래 화살표 | 다음 문장으로 이동 |
| 위 화살표 | 이전 문장으로 이동 |
| Home | 첫 문장으로 이동 |
| End | 마지막 문장으로 이동 |
| Esc | 분석 결과 수정 화면으로 이동 |

## 분석 결과 수정

분석 결과 수정 화면에서는 자동 분석을 교사가 수업에 맞게 바꿀 수 있습니다.

- 성분 역할을 커스텀 드롭다운(select) 메뉴를 통해 변경 (주어~부사어는 단축키 `1`~`6` 및 번호 접두사를 탑재하고 있으며, 제목/소제목은 단축키/접두사 없이 깔끔한 텍스트로 드롭다운 메뉴 목록에 구성)
- 성분 위치를 드래그 앤 드랍으로 이동(성분 사이 빈 자리에 놓으면 순서만 변경)
- 주요소와 종요소 사이 이동
- 원하는 단어만 선택해 분리
- 여러 단어를 선택해 하나의 새 성분으로 분리
- 성분을 다른 성분 위에 겹치게 드래그해 병합(키 없이 동작, 문장 순서대로 합쳐짐)
- 형용사어가 수식하는 원문 단어를 복수 선택(Shift+클릭 연속 선택 지원, 자기 자신의 단어는 선택 대상에서 제외)
- 종요소의 발표 화면(3단계) 순차 공개 우선순위 수동 지정 및 뒷순위 당김 관리 (💡 전구 버튼)

자동 분석은 최종 정답이 아니라 교사용 초안입니다. 복잡한 문장에서는 교사가 수정하는 흐름을 전제로 합니다.

## TXT 저장과 불러오기

`TXT 저장`을 누르면 현재 문장과 분석 결과가 `.txt` 파일로 저장됩니다.

저장 파일에는 다음 정보가 포함됩니다.

- 문장 번호
- 원문
- 성분 ID
- 성분 역할
- 주요소/종요소 줄 정보
- 원문 단어 인덱스
- 종요소 위치
- 형용사어 수식 대상
- 종요소 우선순위 공개 순번 (pinOrder)
- 앱이 복원할 수 있는 JSON 데이터

지원되는 브라우저에서는 저장 위치 선택 창이 뜹니다. 지원하지 않는 브라우저에서는 일반 다운로드 방식으로 저장됩니다.

`TXT 불러오기`를 누르고 저장했던 TXT 파일을 선택하면 문장, 성분, 위치, 분리/병합 결과, 형용사어 수식 대상이 복원됩니다.

### 최근 폴더 기억 및 파일 세션 동기화

- **최근 작업 폴더 기억**: 브라우저의 File System Access API를 활용하여, 지문을 불러오거나 저장할 때 브라우저가 최근에 열었던 디렉토리(`id` 매핑 및 `startIn` 파일 핸들 제공)를 기억하고 다음 탐색창을 켤 때 해당 위치에서 바로 시작합니다.
- **덮어쓰기 파일 정보 즉시 갱신**: "다른 이름으로 저장"을 진행하면 현재 활성화된 파일 정보가 갱신되어, 바로 뒤에 "현재 TXT에 저장"을 눌렀을 때 이전에 저장했던 파일이 아닌, 방금 새로 저장한 파일에 정확하게 덮어쓰기가 수행됩니다.
- **1단계 자동 취합**: 2단계 문장 수정 중 "1단계-대시보드" 버튼을 눌러 이전 화면으로 돌아가더라도 수정된 텍스트가 1단계 지문에 자동으로 취합 반영되어 데이터 유실을 방지합니다.

## 화면 표시 기준

- 글씨체는 `Pretendard`를 사용합니다.
- 주요소는 밝은 색상으로 크게 표시합니다.
- 종요소는 별도 색상과 박스로 표시합니다.
- 주어, 동사, 목적어, 보어, 중요소 색상은 화면 상단 범례로 안내합니다.
- 성분 내부 단어는 줄바꿈되지 않도록 표시합니다.
- 화면이 좁거나 문장이 길면 글자 크기를 자동 조정합니다.
- 종요소는 하나의 성분이 한 줄을 차지합니다.
- 여러 종요소는 공개 순서대로 누적 표시됩니다.

## 남은 확인 사항

최종 버전에서 주요 기능은 구현되었지만, 실제 교실 화면 크기에서는 다음 항목을 수동으로 확인하는 것이 좋습니다.

- `Let's`, `today's` 같은 apostrophe 포함 문장
- 문장 맨 뒤에 붙는 긴 부사구
- 여러 종요소가 같은 anchor 근처에 붙는 문장
- 형용사어가 여러 단어 또는 종요소를 수식하는 문장
- 저장한 TXT 파일을 다시 불러오는 흐름
