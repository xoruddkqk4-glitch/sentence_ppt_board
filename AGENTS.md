# AGENTS.md - whiteboard_ppt 프로젝트 지침

## 프로젝트 개요

`whiteboard_ppt`는 영어 교사가 수업 중 영어 지문을 붙여넣으면, 앱이 지문을 문장 단위로 나누고 학생들이 보기 좋은 발표 화면으로 보여주는 교실용 정적 웹 앱이다.

핵심 사용자는 영어 교사이며, 핵심 시청자는 교실 맨 뒤에 앉은 학생이다. 따라서 모든 화면 설계와 구현은 수업 중 빠른 조작, 큰 가시성, 높은 대비, 명확한 문장 성분 표시를 우선한다.

## 현재 PDCA 상태

- 프로젝트 레벨: `Starter`
- 활성 기능: `sentence-presentation-app`
- 현재 단계: `completed` (Report 완료)
- 완료 단계: `Plan`, `Design`, `Do`, `Check`, `Report`
- 최종 Match Rate: 94% (목표 90% 초과)
- PDCA 반복 횟수: 1회
- 다음 작업: archive (구현·검증·보고 완료, 보관 단계 대기)

주요 문서:

- 계획 문서: `docs/01-plan/features/sentence-presentation-app.plan.md`
- 설계 문서: `docs/02-design/features/sentence-presentation-app.design.md`
- 분석 문서: `docs/03-analysis/sentence-presentation-app.analysis.md`
- 완료 보고서: `docs/04-report/sentence-presentation-app.report.md`
- 루트 디자인 가이드: `DESIGN.md`

주의:

- 기능 요구사항의 1차 기준은 PDCA Plan/Design 문서다.
- 루트의 `DESIGN.md`는 이 프로젝트에 맞춘 디자인 가이드이며 UI 구현 시 함께 참고한다.

## 제품 요구사항 요약

앱은 다음 흐름을 지원해야 한다.

1. 교사가 영어 지문을 입력하거나 붙여넣는다.
2. 앱이 지문을 문장 단위로 나눈다.
3. 앱이 각 문장의 문장 성분 분석 초안을 만든다.
4. 교사가 분석 결과를 드래그앤드롭으로 수정한다.
5. 학생용 발표 화면에서 문장을 하나씩 크게 보여준다.

## 문장 성분 표시 규칙

문장은 주요소와 종요소로 나누어 표시한다.

주요소:

- 주어
- 동사
- 보어
- 목적어

종요소:

- 형용사어
- 부사어

화면 배치:

- 주요소는 맨 윗줄 한 줄에 모두 표시한다. (제목, 소제목 역시 `lane: "major"`로 주요소 줄에 배치된다.)
- 주요소는 성분 라벨을 붙이지 않고 색으로 구분한다.
- 주어, 동사, 목적어, 보어 색상 범례는 발표 화면 위쪽에 표시한다.
- 종요소가 없는 순수 제목/소제목 문장(`isPureTitle`)은 발표 화면 전체 화면 기준 중앙 정렬된 대형 텍스트 블록 형태로 단독 렌더링한다.
- 제목/소제목 문장 내에 종요소(형용사어, 부사어)가 존재하는 경우, 일반 문장과 마찬가지로 하단 영역에 종요소 카드들이 배치되며, 연결선(부사어) 및 수식 화살표와 밑줄(형용사어)이 제목 내 단어들을 깔끔하게 이어서 표현해 준다.
- 종요소는 주요소 아래에 성분 하나당 한 줄로 표시한다.
- 종요소는 라벨 없이 하나의 별도 색으로 통일한다.
- 종요소가 1개면 발표 문장 영역은 총 2줄, 종요소가 2개면 총 3줄이 된다.
- 종요소는 문장 안 원래 위치를 기준으로 해당 줄에서 가운데 정렬한다.
- 같은 문장 성분에 속한 단어 묶음은 중간에서 줄바꿈되지 않아야 한다.
- 화면이 좁아 줄바꿈이 필요하면 성분 묶음 내부가 아니라 성분 묶음 사이에서만 줄바꿈한다.
- 종요소 위치를 가리키는 동그라미 점은 문장 맨 앞, 단어 사이, 문장 맨 뒤에만 놓고 주요소 단어와 겹치지 않는다.
- 여러 종요소의 점은 서로 겹치지 않고 문장 순서대로 배치한다.
- 가장자리 점 여백은 공개된 종요소 수가 아니라 문장의 전체 종요소를 기준으로 잡아, 종요소를 1~2개만 공개해도 첫/마지막 단어와 겹치지 않게 한다. 주요소의 단어 수가 짧아 글자 크기가 큰 경우에는 가장자리 점의 위치를 폰트 크기에 비례하여 더 바깥으로 밀어내어 넓게 여백을 확보한다.
- 종요소 앵커(점/화살표)가 항상 박스의 가로 폭 영역(0%~100%) 내에 위치하도록 박스 Clamping 범위를 제한하고, 박스 상단에서 앵커까지 완벽하게 수직인 정렬선으로 매끄럽게 연결한다.
- 종요소 행 사이 세로 간격은 촘촘하게 유지한다.
- 형용사어는 발표 화면에서 수식 대상에 밑줄과 화살표로 수식 관계를 표시한다.
- 형용사어 화살표는 수식 대상(주요소 포함)을 나타내는 괄호(bracket)의 밑변 정가운데를 정확히 가리켜야 한다.
- 형용사어는 자기 자신의 단어를 수식 대상으로 삼지 않는다.

## 교사 보정 기능

자동 분석은 틀릴 수 있으므로 최종 답이 아니라 교사용 초안으로 다룬다.

필수 동작:

- 교사는 성분 묶음을 드래그앤드롭으로 이동할 수 있어야 한다.
- 같은 줄 안에서 성분 순서를 바꿀 수 있어야 한다.
- 주요소 줄과 종요소 줄 사이로 성분을 옮길 수 있어야 한다.
- 성분 사이 빈 자리에 놓으면 순서만 바꾸고, 다른 성분 위에 겹치게 놓으면 별도 키 없이 병합한다.
- 병합한 성분의 텍스트는 항상 문장 순서대로 합쳐져야 한다.
- 교사는 잘못 분류된 성분 역할을 수정할 수 있어야 한다. (성분 칩의 드롭다운을 통해 주어~부사어는 단축키 `1`~`6`번 및 번호 접두사를 활용해 신속히 교체할 수 있고, 제목 및 소제목은 단축키와 번호 접두사 없이 깔끔한 텍스트로 드롭다운 메뉴에 노출된다.)
- 드래그 중에도 성분 묶음 내부 단어는 분리되거나 줄바꿈되지 않아야 한다.
- 수정 결과는 현재 발표 세션 안에서 유지되어야 하며, 로컬 스토리지 캐시(`sentence_board_file_cache_${fileName}`)에 실시간 연동되어 빠른 지문 불러오기를 실행하더라도 수동 불러오기와 완벽히 일치하는 수정본이 복원되어야 한다.

권장 구현 방향:

- 수정 화면과 발표 화면을 분리한다.
- 드래그앤드롭은 수정 화면에서만 활성화한다.
- 발표 화면에서는 주요소는 고정하고, 종요소 네모칸만 드래그앤드롭으로 삽입 위치를 조정할 수 있게 한다.
- 종요소 위치는 서로 영향을 주지 않고 각각 독립적으로 이동되어야 한다.
- 발표 화면 마지막 문장에서 "이전 단계" 버튼은 발표를 끝내고 2단계(수정) 화면으로 돌아간다.
- 발표 화면 오른쪽 위 "전체화면" 버튼은 Fullscreen API로 전체화면을 토글한다.
- 전체화면에서도 발표 화면 배경은 평소와 동일하게 유지해, 전자칠판 판서가 가능하도록 한다(검정 배경 금지).
- 전자칠판 터치를 고려해 드래그 핸들과 조작 버튼은 크게 만든다.
- "기본 폴더 설정" 버튼을 통해 수업용 로컬 지문 폴더를 연동하고, 5x5 그리드의 "빠른 지문 불러오기" 버튼으로 손쉽게 연동하여 불러오며, 불러온 분석 데이터는 수동 "TXT 불러오기"를 통해 읽은 내용과 완벽한 일관성을 갖는다. 또한 브라우저 재실행(새로고침) 시에도 연동했던 폴더와 지문 목록 및 콘텐츠가 로컬 스토리지 백업 캐시를 바탕으로 완전 자동 복구된다.
- 2단계 수정 화면의 "중요 문장 지정" 버튼은 파스텔 노랑색으로 눈에 띄게 시각화한다.
- 3단계 발표 화면 왼쪽 사이드바에 문장 번호들이 동적 배경(기본: 회색, 중요: 노랑/골드, 활성: 브랜드 블루, 활성+중요: 골드 테두리가 둘러진 브랜드 블루)을 갖는 둥근 카드 단추 형태로 세로 배열된다.
- 사이드바 내 문장 번호는 단축키 배지 유무와 상관없이 버튼 가로 폭의 정가운데에 일관되게 정렬되며, 좌측의 미니 별표(☆/★)를 클릭해 발표 화면에서도 중요도를 즉시 토글할 수 있다.
- 중요 문장으로 지정된 문장들은 지문 내 등장 순서대로 숫자 `1`~`9` 단축키에 매핑(이가 빠지는 빈자리 없이 자동 압축 및 재정렬)되며, 키보드 숫자를 누르거나 사이드바 번호 카드를 클릭하면 해당 문장으로 바로 점프(이동)한다.
- 교사는 2단계 수정 화면의 각 종요소 칩 우측에 있는 전구(💡) 버튼을 눌러 3단계 발표 화면에서의 순차 공개 우선순위(💡 1, 💡 2, 💡 3...)를 수동 지정할 수 있다. 중간 번호를 해제하면 뒷순위 번호들이 자동으로 1씩 당겨져(💡 3 ➡️ 💡 2) 번호가 끊김 없이 압축된다. 2단계에서는 원문 순서가 유지되나, 3단계 발표 화면에서는 이 우선순위 💡 순서대로 정렬되어 순차적으로 공개된다.

## 데이터 모델 기준

앱 상태는 단순한 JavaScript 객체로 관리한다.

```js
const state = {
  passageText: "",
  sentences: [],
  currentSentenceIndex: 0,
  mode: "input",
  settings: {
    fontScale: 1,
    theme: "light",
    align: "center"
  }
};
```

각 문장 성분 묶음은 다음 정보를 가져야 한다.

```js
const component = {
  id: "s1-c1",                 // 문장 안에서 반드시 고유해야 한다
  text: "The young students",
  role: "subject",
  lane: "major",
  order: 1,
  startIndex: 0,               // 원문 전체 단어 기준 시작 인덱스
  endIndex: 3,                 // 원문 전체 단어 기준 끝 인덱스(미포함)
  modifierTargetIndexes: [],   // 형용사어가 수식하는 원문 단어 인덱스 목록
  pinnedAnchor: false,         // 발표 화면에서 위치를 수동 고정했는지
  boxPercent: undefined,       // 발표 화면에서 박스를 수동 이동한 위치(%)
  pinOrder: undefined          // 발표 화면(3단계)에서 종요소 공개 우선순위 (💡 1, 💡 2...)
};
```

성분 `id`는 한 문장 안에서 반드시 고유해야 한다. 분리/병합/문장 재인덱싱 시 id가 중복되면 발표 화면의 위치 매칭과 수정 화면의 성분 선택이 서로 충돌하므로, id는 순번 기반으로 새로 부여하고 저장 파일을 불러올 때도 고유 id를 다시 만든다. 형용사어의 수식 대상은 id가 아니라 원문 단어 인덱스(`modifierTargetIndexes`)로 표현해 id가 바뀌어도 복원되게 한다.

`role` 값:

- `subject`
- `verb`
- `complement`
- `object`
- `adjective`
- `adverb`
- `title`
- `subtitle`

`lane` 값:

- `major`
- `minor`

## 디자인 지침

가시성:

- 교실 맨 뒤 학생도 읽을 수 있도록 발표 문장을 매우 크게 표시한다.
- 장식보다 글자 크기, 대비, 여백, 줄 간격을 우선한다.
- 발표 화면은 화면 전체를 넓게 사용한다.
- 발표 문장을 작은 카드 안에 가두지 않는다.
- 발표 화면에서 성분 요소(칩) 클릭 시 해당 요소만 전체 화면에 크게 보여주는 확대 화면(오버레이)을 지원하며, 배경색은 발표 화면의 테마(밝음/어두움)와 완벽히 동기화된다.
- 확대 화면 내 텍스트는 한 줄에 최대 10단어씩 나누어 배치하고, 모두 왼쪽 정렬한다.
- 확대 화면 내 각 줄별 줄간격(line-height)은 판서용 공간 확보를 위해 기존(2.7)의 2배인 5.4로 넓게 설정한다.
- 확대 화면의 텍스트가 화면 가로 길이를 최대한 꽉 채울 수 있도록 글자 크기를 양방향(확대/축소)으로 동적 스케일링한다. 가로 한계선(window.innerWidth - 120)과 세로 한계선(window.innerHeight - 200)을 충족하는 범위 내에서 98% 비율로 꽉 차게 피팅된다.

글씨체:

- 전체 UI와 발표 화면에는 `Pretendard`를 사용한다.
- CSS 폰트 스택은 `"Pretendard", system-ui, sans-serif`를 기본으로 둔다.
- 폰트 로딩 실패에 대비해 반드시 대체 글꼴을 둔다.

줄바꿈:

- 성분 묶음에는 `white-space: nowrap` 계열 처리를 적용한다.
- 성분 줄 컨테이너는 `flex-wrap`을 허용하되, 묶음 내부가 아니라 묶음 사이에서만 줄바꿈되게 한다.

예상 CSS 방향:

```css
.component-lane {
  display: flex;
  flex-wrap: wrap;
  justify-content: center;
  gap: var(--component-gap);
}

.component-chip {
  display: inline-flex;
  align-items: baseline;
  white-space: nowrap;
}
```

## 구현 규칙

- 문서는 한글로 작성한다.
- HTML은 구조, CSS는 모양, JavaScript는 동작을 담당하게 한다.
- 파일 이름과 함수 이름은 명확하게 짓는다.
- 함수는 작게 유지한다.
- 불필요한 라이브러리를 추가하지 않는다.
- 첫 구현에서는 외부 패키지 없이 브라우저 기본 기능으로 구현한다.
- 인터넷이 없어도 기본 동작이 가능한 구조를 선호한다.
- 기존 사용자 변경사항을 되돌리지 않는다.

## 주요 함수 후보

- `splitSentences(text)`: 지문을 문장 배열로 나눈다.
- `analyzeSentence(sentenceText)`: 문장 성분 초안을 만든다.
- `renderInputView()`: 입력 화면을 그린다.
- `renderEditView()`: 수정 화면을 그린다.
- `renderPresentView()`: 발표 화면을 그린다.
- `moveComponent(componentId, targetLane, targetIndex)`: 성분 묶음을 이동한다.
- `updateComponentRole(componentId, role)`: 성분 역할을 변경한다.
- `goToSentence(index)`: 현재 문장 번호를 바꾼다.
- `applySettings(settings)`: 글자 크기, 테마, 정렬을 반영한다.

## 수동 QA 기준

구현 후 최소한 다음을 확인한다.

- 빈 입력에서 안내 메시지가 보인다.
- 일반 영어 지문이 문장 단위로 나뉜다.
- 주요소는 윗줄에 크고 굵게 표시된다.
- 종요소는 아랫줄에 약간 작은 글씨로 표시된다.
- 같은 성분 묶음 내부에서 줄바꿈이 일어나지 않는다.
- 성분 묶음을 같은 줄 안에서 드래그해 순서를 바꿀 수 있다.
- 성분 묶음을 주요소 줄과 종요소 줄 사이로 옮길 수 있다.
- 성분 역할을 수정하면 화면이 즉시 갱신된다.
- 이전/다음 문장 이동이 작동한다.
- 키보드 조작이 작동한다.
- 글자 크기와 테마 변경이 발표 화면에 반영된다.
